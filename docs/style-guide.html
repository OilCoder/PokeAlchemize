Filosofía de diseño

Fundamento

PokeAIchemize es el cruce entre una investigación de IA generativa y el universo Pokémon. El diseño debe reflejar exactamente esa dualidad: precisión técnica + magia elemental. No es una web comercial ni un portfolio — es un laboratorio con estética de codex.

⚗️
Alquimia
La transformación es el concepto central. Cada decisión visual debe evocar transmutación: antes/después, original/reimaginado. El acento cian representa la energía del proceso.
🔬
Precisión técnica
El pipeline es complejo y real. El UI debe reflejarlo: monospace para datos, jerarquía clara, densidad de información controlada. No hay decoración vacía.
✨
18 tipos = 18 paletas
Los tipos elementales son el sistema de color vivo del proyecto. Fuego, Agua, Dragón — cada tipo tiene su vocabulario visual completo. Son el alma cromática de todo.
/* Principios que guían cada decisión */ 1. Oscuro como base → el contenido brillará sobre él 2. Acento único (#3aa4ff) → no inventar colores nuevos sin razón 3. Los tipos son color → usarlos para accents, no como decoración 4. Monospace para datos, Display para jerarquía, Sans para cuerpo 5. Hover eleva, Active ilumina → dos estados bien diferenciados 6. Sin emojis en UI → solo en iconos de tipo (TYPE_SYSTEM.icon) 7. Menos es más → no añadir secciones sin contenido real

Layout

Grid de 3 columnas + header fijo de 56px. Las vistas full-width ocultan sidebar y rightbar añadiendo clase al .app.

/* Grid principal */ .app { display: grid; grid-template-rows: 56px 1fr; grid-template-columns: 260px 1fr 340px; grid-template-areas: "logo header header" "sidebar main rightbar"; height: 100vh; } /* Vista full-width (About, Style Guide, etc.) */ .app.view-about .sidebar, .app.view-about .rightbar, .app.view-about #main { display: none; } .app.view-about .about-view { display: block; grid-column: 1 / -1; /* span all columns */ } /* Siempre usar flex/grid+gap — nunca márgenes en hijos */ .sg-row { display: flex; gap: 14px; } .sg-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(200px,1fr)); gap: 14px; }

Voz y tono

La UI habla en español neutro técnico. El lore usa tono descriptivo-científico, como una enciclopedia de criaturas reales.

SÍ · Hacer
  • Toda la UI en español neutro
  • Labels en MAYÚSCULAS: POKÉDEX, FASE A
  • Nombres de pokémon en caps: BULBASAUR
  • Números de Pokémon: #001 (3 dígitos con cero)
  • Tono científico en lore: "La piel adquiere..."
  • Archivos en minúsculas snake_case
  • Énfasis con negritas, no cursivas
NO · Evitar
  • Mezclar español e inglés en la misma frase de UI
  • Datos ficticios o placeholders Lorem ipsum
  • Labels más de 3-4 palabras
  • Punto final en títulos y labels
  • Emojis en UI (solo en type-chip icon)
  • Texto genérico "Cargando...", "Sin datos"
  • IA hablando en primera persona en el lore
