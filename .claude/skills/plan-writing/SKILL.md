# Plan Writing

## Procedure

1. Read `todo/PLAN.md` and any recent `todo/bitacora-*.md` files.
2. Review `git log` for recent commits.
3. If creating: ask for goal/stack/phases, draft, show draft, wait for approval, then save.
4. If updating: mark completed tasks `[x] (YYYY-MM-DD)`, add approved new ones only.

## Rules

- Flat checkboxes, independent phases, mark immediately, never delete tasks.
- Always show draft and wait for approval before saving.
- Check bitacora `- [ ]` items as candidates for the plan.
- Follow format defined in `plan-format.md`.
