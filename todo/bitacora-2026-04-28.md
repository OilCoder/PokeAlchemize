### Design decisions
- `web/` usa paths `../data/` y `../outputs/` para assets reales, pero `data/bundle.json` y `outputs/moves/` viven dentro de `web/` (son propios del frontend, no del pipeline)
- `bundle.json` se regenera después de cada run del pipeline con `PYTHONPATH=. python3 pipeline/10_bundle_builder.py`
- El background se pone DESPUÉS de "Full body portrait, centered character" para que la composición tenga prioridad sobre el escenario en el budget de 77 tokens CLIP
- Se decidió borrar todos los prompts existentes al añadir el background (necesario para que E3 los regenere)

### Learnings
- El proceso del pipeline que corre en Docker carga `config.py` al inicio — cambios en el host no afectan el proceso ya en marcha; hay que matarlo y relanzar para que tome la nueva configuración
- Los archivos en `outputs/` son de `root` (creados dentro del container) → para borrarlos desde el host hace falta `docker exec` o `sudo`
- El `run.log` acumula entradas de múltiples ejecuciones del pipeline porque ambos (proceso interno y `tee`) escriben al mismo archivo montado en el volumen
- `bundle.json` en el handoff tiene `pokemonMeta` con los outputs de E1 (no de combo_data)
