### Design decisions
- [Quitar "Color palette:" label] — Z-Image fue entrenado con imágenes que incluyen
  tarjetas de referencia de paleta; la etiqueta dispara ese comportamiento de forma
  estocástica (más frecuente en dark type con paleta muy contrastante)
- [anchor_phrases[0] en el name_line] — usar solo el primer anchor (más distintivo)
  mantiene el presupuesto de tokens bajo; meter la lista completa empujaría skin_material
  y signature_feature fuera del window de 77 tokens
- [77-token budget layout] — orden fijo: colores (~12) → nombre+tipo+anchor (~12) →
  Ken Sugimori style (~12) → skin_material (~15) → signature_feature (~15) → accent (~6);
  total crítico ~72 tokens, accent cae en el borde pero es el elemento menos importante
- [Prompt narrativo vs estructurado] — experimento Charizard agua confirmó que los
  prompts narrativos ("No fire, no orange") no suprimen colores canónicos en Z-Image
  distilled; el formato estructurado palette-first (variantes A y C) suprime mejor
- [No cambiar más] — usuario revisó resultados del Charizard y decidió mantener el
  formato actual del pipeline sin más iteraciones de prompt

### Learnings
- Z-Image no suprime colores canónicos con instrucciones explícitas ("No fire") —
  solo funciona desplazarlos poniendo los colores deseados primero en el prompt
- `net use Z: \\wsl.localhost\...` falla con Error 64 aunque la ruta sea accesible;
  `pushd \\wsl.localhost\Ubuntu-24.04\home\pokinux\PokeAIchemize` funciona porque
  no crea un share SMB persistente
- Workflow definido: WSL Claude Code = pipeline de generación; Windows Claude Desktop
  = desarrollo del frontend web; ambos sobre el mismo filesystem WSL vía `pushd`
- La distro WSL se llama `Ubuntu-24.04` (no `Ubuntu`) — relevante para rutas UNC
